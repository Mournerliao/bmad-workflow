# BMAD Help

Use this skill when the user wants orientation on the BMAD workflow itself.

## Explain

- the workflow path: `idea -> clarify -> plan -> build -> review`
- the available entry points: `bmad-help`, `bmad-quick-dev`, `bmad-plan`, `bmad-review`
- the main outputs: `brief`, `plan`, `solution-outline`, `build-checklist`, `review-report`
- the role model: `analyst`, `planner`, `architect`, `builder`, `reviewer`
