# Capability: Plan

## Purpose

Clarify a request and produce an execution plan plus a lightweight technical direction before implementation starts.

## Inputs

- a feature or change request that needs planning
- ambiguous work that needs scope clarification

## Outputs

- `brief`
- `plan`
- `solution-outline`

## Workflow Mapping

This capability should orchestrate these roles in order:

1. `analyst`
2. `planner`
3. `architect`

It should stop before build or review.
