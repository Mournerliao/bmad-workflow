# Capability: Review

## Purpose

Review a plan, patch, or implementation for risks, regressions, and validation gaps.

## Inputs

- an implementation summary
- a patch or code change
- earlier workflow artifacts such as `brief`, `plan`, or `solution-outline`

## Outputs

- a structured review result
- findings, regression watchlist, validation gaps, and next-step guidance

## Workflow Mapping

This capability is reviewer-led and should center the `reviewer` role.
