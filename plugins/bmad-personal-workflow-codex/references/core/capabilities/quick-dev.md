# Capability: Quick Dev

## Purpose

Run the main BMAD personal workflow from problem framing through review.

## Inputs

- a feature request
- a bugfix request
- a scoped refactor or implementation task

## Outputs

- `brief`
- `plan`
- `solution-outline`
- `build-checklist`
- `review-report`

## Workflow Mapping

This capability maps to the `quick-dev` workflow and should orchestrate these roles in order:

1. `analyst`
2. `planner`
3. `architect`
4. `builder`
5. `reviewer`
